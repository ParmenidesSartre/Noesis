## 6.1 Assignment Creation and Submission

### 6.1.1 Assignment Creation by Teachers

**Creating Assignments:**
- Assignment form:
  - Assignment title
  - Subject/class selection
  - Assignment description (rich text editor):
    - Text formatting
    - Insert images
    - Insert links
    - File attachments
  - Assignment type:
    - Homework
    - Project
    - Essay
    - Worksheet
    - Research
    - Practical/Lab work
    - Group assignment
    - Individual assignment
  - Due date and time
  - Total marks/points
  - Weightage (% towards final grade)
  - Difficulty level (Easy/Medium/Hard)
  - Estimated time to complete

**Assignment Attachments:**
- Upload assignment files:
  - PDF worksheets
  - Word documents
  - Images
  - Reference materials
  - Links to external resources
- Maximum file size (e.g., 25MB per file)
- Multiple files allowed
- File format restrictions (if any)

**Assignment Settings:**
- Submission method:
  - Online upload
  - In-class submission (physical)
  - Both
- Allow late submissions (Yes/No)
  - Late penalty (X% deduction per day late)
  - Maximum days late accepted
- Resubmission allowed (Yes/No)
  - Number of attempts allowed
- Peer review enabled (students review each other's work)
- Plagiarism check enabled (integration with plagiarism tools)

**Publishing:**
- Save as draft (not visible to students)
- Publish immediately
- Schedule publish date/time
- Notify students:
  - Email notification
  - Push notification (mobile app)
  - In-app announcement

### 6.1.2 Student Assignment Submission

**Viewing Assignments:**
- Students see all pending assignments
- Filter by:
  - Subject
  - Status (pending/submitted/graded)
  - Due date
- Sort by due date (nearest first)
- Calendar view of all assignments
- Overdue assignments highlighted in red

**Submission Process:**
- Click on assignment
- View assignment details and instructions
- Download attached files/worksheets
- Work on assignment (offline)
- Upload completed work:
  - File upload (PDF, Word, images, etc.)
  - Text entry (for essays/short answers)
  - Link submission (for online projects)
- Add submission notes/comments
- Preview submission
- Submit assignment:
  - Confirmation prompt
  - Submission timestamp recorded
  - Confirmation receipt email

**Late Submission:**
- System shows "LATE" badge if submitted after due date
- Days/hours late calculated automatically
- Late penalty applied (if configured)
- Student can see penalty deduction

**Resubmission:**
- If allowed, student can delete and resubmit
- Previous submission versioning (all versions saved)
- Teacher can view all submission attempts

### 6.1.3 Assignment Tracking

**Teacher View:**
- See all assignments created
- Submission status:
  - Total students
  - Submitted count
  - Pending count
  - Overdue count
  - Submission rate (%)
- List of students who submitted vs not submitted
- Downloadable submission files (bulk download)
- Export submission list to Excel

**Student View:**
- See all assigned assignments
- Submission history
- Graded assignments with scores
- Feedback from teacher
- Average assignment score

**Parent View:**
- View child's assignments
- See submission status
- View grades and feedback
- Receive notifications for:
  - New assignments posted
  - Assignments due soon
  - Assignments overdue
  - Assignments graded

**Assignment Reminders:**
- Auto-reminder to students:
  - 3 days before due
  - 1 day before due
  - On due date
  - If overdue
- Auto-reminder to teacher if many students haven't submitted

## 6.2 Quiz and Test Management

### 6.2.1 Creating Quizzes/Tests

**Quiz Builder:**
- Quiz/Test details:
  - Title
  - Subject/class
  - Description/instructions
  - Total marks
  - Duration (time limit)
  - Number of questions
  - Pass marks (minimum to pass)
  - Difficulty level

**Question Types:**
- Multiple choice (single answer)
- Multiple choice (multiple answers/checkboxes)
- True/False
- Short answer (text input)
- Essay/Long answer
- Fill in the blanks
- Matching
- Ordering/Sequencing
- File upload (for practical assessments)

**Question Bank:**
- Create and save questions to bank
- Categorize by topic/chapter
- Tag questions (e.g., "algebra", "basic", "PSLE")
- Reuse questions across quizzes
- Import questions from Excel/CSV
- Random question selection from bank

**Quiz Settings:**
- Test type:
  - Practice quiz (formative, low stakes)
  - Graded quiz (summative, counts towards grade)
  - Mock exam
  - Diagnostic test
- Availability:
  - Start date/time
  - End date/time (deadline)
  - Or always available (practice mode)
- Attempt limit:
  - Unlimited attempts (practice)
  - Single attempt (formal test)
  - X attempts allowed
- Time limit:
  - No time limit
  - X minutes per attempt
  - Timer displayed to student
- Question display:
  - All questions on one page
  - One question per page
  - Random order (prevent cheating)
- Feedback settings:
  - Show correct answers immediately after submission
  - Show correct answers after deadline
  - Never show correct answers (teacher explains in class)
  - Show score immediately or after teacher reviews

**Quiz Security:**
- Prevent tab switching (flags if student leaves quiz page)
- Disable right-click/copy-paste
- Randomize question order
- Randomize answer options
- IP address logging (detect multiple users)
- Proctor mode (require teacher supervision for online tests)

### 6.2.2 Taking Quizzes/Tests (Student View)

**Starting Quiz:**
- Student clicks on quiz
- See instructions and rules
- See time limit and number of questions
- "Start Quiz" button
- Timer starts upon starting

**Quiz Interface:**
- Question displayed with options
- Navigation:
  - Next question
  - Previous question
  - Jump to specific question number
- Mark question for review (revisit later)
- Timer countdown displayed
- Auto-save responses (in case of connection loss)

**Submitting Quiz:**
- Review all answers before submitting
- List of unanswered questions highlighted
- Warning if time running out (5 min, 1 min)
- Auto-submit when time expires
- Confirmation prompt: "Are you sure you want to submit?"
- Submission timestamp

**Results:**
- If instant results enabled:
  - Score displayed
  - Correct vs incorrect count
  - Percentage score
  - Pass/fail status
  - Review answers (if allowed)
- If results hidden:
  - Confirmation of submission only
  - Wait for teacher to grade/release results

### 6.2.3 Grading and Results

**Auto-Grading:**
- Multiple choice, True/False, Fill-in-blanks auto-graded
- Score calculated immediately
- Partial credit (for multiple answer questions)

**Manual Grading:**
- Short answer and essay questions require teacher review
- Teacher views student responses
- Assign marks per question
- Add feedback comments
- Save and continue (draft grading)
- Finalize and release grades

**Grade Management:**
- View all student scores for a quiz
- Grade distribution (histogram)
- Average score
- Highest and lowest scores
- Identify struggling students
- Export results to Excel

**Feedback:**
- Provide individual feedback per student
- General feedback for the class
- Feedback on specific questions
- Recommendations for improvement

**Result Release:**
- Release results to all students at once
- Release individually as graded
- Schedule release date/time
- Notify students when results available

## 6.3 Grade Recording and Tracking

### 6.3.1 Gradebook

**Gradebook Structure:**
- Class-based gradebook
- Rows: Students
- Columns: Assessments (assignments, quizzes, tests, exams)
- Each cell: Grade/score

**Grade Entry:**
- Manual grade entry by teacher
- Import grades from Excel
- Auto-populated from quizzes (auto-graded)
- Bulk grade entry (enter same grade for multiple students)
- Quick edit mode

**Assessment Categories:**
- Organize assessments by category:
  - Homework (X% of final grade)
  - Quizzes (Y% of final grade)
  - Tests (Z% of final grade)
  - Participation (W% of final grade)
  - Final Exam (V% of final grade)
- Weightage configuration
- Drop lowest score policy (e.g., drop 1 lowest homework score)

**Grading Scale:**
- Percentage (0-100%)
- Letter grades (A, B, C, D, F)
  - Customizable ranges (e.g., A: 90-100%, B: 80-89%)
- GPA (e.g., 4.0 scale)
- Custom grading scales

**Missing/Incomplete Grades:**
- Mark as "Not Submitted" or "Incomplete"
- Exclude from average calculation or count as zero
- Flag students with missing grades
- Remind teacher to enter missing grades

### 6.3.2 Grade Calculation

**Weighted Average:**
- Calculate final grade based on weighted categories
- Formula: (Homework × 20%) + (Quizzes × 30%) + (Tests × 30%) + (Final Exam × 20%)
- Customizable formula per class

**Extra Credit:**
- Allow extra credit assignments
- Add to total score (can exceed 100%)
- Or separate extra credit category

**Curved Grading:**
- Apply curve to final grades (if enabled)
- Adjust all scores up by X points
- Or scale to fit desired distribution

**Pass/Fail Criteria:**
- Set minimum passing grade (e.g., 50%, 60%, C)
- Students below threshold marked as "Fail"
- Alert student and parent

**Grade Comments:**
- Add comments to individual grades
- Explain deductions or bonuses
- Provide feedback

### 6.3.3 Grade Visibility and Access

**Student View:**
- View own grades only
- See individual assessment scores
- See category averages
- See final/current grade
- Track progress over time (grade trend graph)

**Parent View:**
- View child's grades
- Same details as student view
- Historical grade data
- Receive grade notifications:
  - New grade posted
  - Grade dropped below threshold
  - Final grade published

**Teacher View:**
- View entire class gradebook
- Edit grades
- Generate reports
- Analyze class performance

**Admin View:**
- View all classes across branch
- Monitor grade entry progress (teachers behind on grading)
- Flag failing students
- Generate academic reports

**Grade Privacy:**
- Grades are confidential
- Students cannot see each other's grades
- Option to display class average/median (anonymized)

## 6.4 Progress Reports

### 6.4.1 Report Types

**Weekly Progress Report:**
- Summary of week's activities:
  - Classes attended
  - Assignments submitted
  - Quizzes/tests taken
  - Participation notes
- Automated email to parents every Sunday/Monday
- Brief and concise

**Monthly Progress Report:**
- More detailed than weekly
- Subject-by-subject breakdown
- Attendance summary
- Grade summary (all assessments)
- Strengths and weaknesses identified
- Teacher comments (brief)
- Recommended actions
- Generated automatically end of each month
- PDF format, emailed to parents

**Mid-Term Report:**
- Comprehensive report at mid-point of term
- Formal assessment of progress
- Detailed grades for all assessments
- Attendance record
- Behavior and participation notes
- Teacher feedback (detailed)
- Areas needing improvement
- Parent meeting recommended (if needed)

**End-of-Term Report (Report Card):**
- Most comprehensive and formal
- Final grades for all subjects
- Cumulative GPA
- Attendance summary
- Conduct grade
- Teacher comments
- Recommendations for next term
- Achievements and awards
- Principal/Admin signature
- Parent signature required (acknowledgment)
- Sent via email + available for download + printed copy (optional)

### 6.4.2 Report Content and Format

**Student Information Section:**
- Student name, ID, photo
- Grade/level
- Class/section
- Term/semester
- Report date

**Academic Performance Section:**
- Subject-wise performance:
  - Subject name
  - Final grade/score
  - Letter grade
  - Class average (for comparison)
  - Grade distribution graph
  - Teacher name
- Overall GPA/average
- Rank in class (optional)
- Honor roll status (if applicable)

**Attendance Section:**
- Total days/sessions in term
- Days/sessions attended
- Days absent
- Days late
- Attendance percentage
- Comparison with previous terms

**Conduct and Behavior:**
- Participation rating
- Behavior rating
- Work ethic rating
- Cooperation rating
- Ratings scale (e.g., Excellent / Good / Satisfactory / Needs Improvement)

**Teacher Comments:**
- Strengths
- Areas for improvement
- Effort level
- Recommendations for parents
- Personalized feedback

**Goals and Objectives:**
- Goals set for next term
- Action plan
- Recommended focus areas

### 6.4.3 Report Generation and Distribution

**Automated Generation:**
- System auto-generates reports based on data
- Pre-fill all data (grades, attendance, etc.)
- Teacher adds personalized comments
- Admin reviews and approves
- Batch generation for entire class/branch

**Customization:**
- Multiple report templates
- Customize per grade level or branch
- Branding (logo, colors)
- Multi-language support

**Distribution:**
- Email to parents (PDF attachment)
- Available in parent portal
- Print copies (if requested)
- SMS notification that report is ready
- Hand out in person (parent-teacher meeting)

**Parent Acknowledgment:**
- Parents must acknowledge receipt
- Digital signature or checkbox in portal
- Track which parents have acknowledged
- Follow-up with parents who haven't acknowledged

## 6.5 Report Card Generation

### 6.5.1 Report Card Design

**Official Report Card:**
- Professional, formal design
- Centre logo and letterhead
- Security features (watermark, hologram for printed version)
- Student photo
- QR code (for verification/authenticity)

**Report Card Sections:**
- Header:
  - Centre name and logo
  - "Official Report Card" title
  - Academic term/year
  - Student name, ID, class, photo
  - Date of issue

- Academic Performance Table:
  - Columns: Subject, Score, Grade, Teacher, Remarks
  - Rows: Each subject
  - Footer: Overall GPA, Cumulative GPA, Rank

- Grading Scale Legend:
  - Explain grading system (A=90-100, B=80-89, etc.)

- Attendance Record:
  - Total sessions, attended, absent, late
  - Attendance %

- Conduct Assessment:
  - Behavior, Participation, Work Ethic
  - Ratings

- Teacher Comments:
  - Form teacher / Class teacher comments
  - Subject teacher comments (optional)

- Signatures:
  - Teacher signature
  - Principal/Centre Director signature
  - Date

- Parent Section:
  - Parent comments (optional)
  - Parent signature
  - Date

**Report Card Formats:**
- PDF (digital, email, downloadable)
- Printed version (official, sealed)
- Transcript format (for formal records)

### 6.5.2 Report Card Approval Workflow

**Draft Report Cards:**
- Teacher creates draft report card
- Enters all grades and comments
- Preview before submission

**Review and Approval:**
- Submit for admin review
- Admin checks:
  - Grade accuracy
  - Complete data entry
  - Appropriate comments
  - No errors or typos
- Admin can:
  - Approve (moves to final)
  - Request changes (back to teacher with notes)
  - Reject (with reason)

**Finalization:**
- Once approved, report card is finalized
- Locked (cannot be edited without admin unlock)
- Ready for distribution

**Bulk Approval:**
- Admin can approve multiple report cards at once
- Filter by class, teacher, status
- Batch approval for efficiency

### 6.5.3 Report Card Distribution

**Digital Distribution:**
- Email PDF to parents
- Available in parent portal for download
- Secure access (requires login)
- Download history tracked

**Physical Distribution:**
- Print official copies
- Seal in envelope
- Distribute at parent-teacher meeting
- Or mail to home address

**Notification:**
- Notify parents when report card is ready
- Email, SMS, push notification
- Reminder to acknowledge receipt

**Historical Records:**
- All report cards archived
- Accessible anytime by student/parent
- Official transcript generation (combines all terms)

## 6.6 Homework Tracking

### 6.6.1 Homework Assignment

**Creating Homework:**
- Similar to assignment creation (see 6.1)
- Specifically tagged as "Homework"
- Usually shorter, more frequent than regular assignments
- Daily or weekly homework

**Homework Schedule:**
- Teachers can schedule recurring homework:
  - Daily (every school day)
  - Weekly (every Monday)
  - Custom schedule
- Template homework (same format, different content)

**Homework Types:**
- Practice worksheets
- Reading assignments
- Problem sets
- Review exercises
- Preparation for next class

### 6.6.2 Homework Submission and Checking

**Submission:**
- Students submit online (file upload or text entry)
- Or mark as "Completed" (for physical homework checked in class)
- Parents can also mark homework complete (for younger students)

**Checking:**
- Teacher marks homework as:
  - Completed (✓)
  - Incomplete (X)
  - Not submitted (-)
- Can assign score/grade (if graded homework)
- Can add feedback comments
- Quick check mode (just mark complete/incomplete, no detailed grading)

**Homework Completion Rate:**
- Track % of homework completed per student
- Homework compliance report
- Identify students consistently missing homework

### 6.6.3 Homework Monitoring (Parent View)

**Parent Dashboard:**
- See all pending homework
- See upcoming homework
- See completed homework
- Homework calendar view

**Homework Reminders:**
- Reminder to student and parent:
  - When homework assigned
  - Due date approaching
  - Overdue homework
- Configurable reminder preferences

**Homework Accountability:**
- Parents can add notes (e.g., "completed at 6 PM")
- Parents can mark as "Need Help" to alert teacher
- Parents receive weekly homework completion summary

**Homework Concerns:**
- If student consistently not completing homework:
  - Alert parent
  - Alert teacher
  - Trigger parent-teacher meeting
  - Investigate reasons (too difficult, too much, lack of understanding)

## 6.7 Curriculum Planning

### 6.7.1 Curriculum Management

**Curriculum Structure:**
- Define curriculum per subject and grade level
- Break down into:
  - Units/Modules
  - Chapters/Topics
  - Learning Objectives
  - Skills to be developed
- Align with national standards (if applicable)

**Curriculum Repository:**
- Store curriculum documents
- Syllabus files (PDF, Word)
- Scope and sequence
- Pacing guide (what to teach when)
- Learning outcomes checklist

**Curriculum Mapping:**
- Map curriculum to assessments
- Ensure all topics are covered
- Track progress through curriculum
- Identify gaps

### 6.7.2 Lesson Planning

**Lesson Plan Template:**
- Lesson title
- Subject and grade
- Date
- Duration
- Topic/Unit
- Learning objectives (what students will learn)
- Materials needed
- Lesson activities:
  - Introduction/Hook
  - Instruction/Explanation
  - Guided practice
  - Independent practice
  - Closure/Summary
- Assessment (how to check understanding)
- Homework assignment
- Notes/Reflections (teacher fills after lesson)

**Lesson Plan Library:**
- Teachers create and save lesson plans
- Share lesson plans with other teachers (if allowed)
- Reuse and adapt previous lesson plans
- Search by topic, grade, subject

**Lesson Plan Review:**
- Admin can review teachers' lesson plans
- Ensure curriculum coverage
- Quality assurance
- Provide feedback to teachers

### 6.7.3 Curriculum Tracking

**Tracking Coverage:**
- Mark topics as "Taught" as lessons are completed
- Visual progress tracker (% of curriculum covered)
- Identify topics not yet taught
- Ensure all topics covered before term end

**Pacing Guidance:**
- Suggested timeline for each topic
- Alert if falling behind schedule
- Adjust pacing as needed

**Alignment with Assessments:**
- Ensure tests cover taught material
- Don't test on material not yet covered
- Review and adjust as needed
